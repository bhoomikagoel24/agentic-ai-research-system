01 — Problem

Literature review is not
an information retrieval problem.

It requires understanding relationships across papers — comparing methods, detecting contradictions, surfacing gaps, and synthesizing findings into a coherent whole.

Traditional LLM systems can summarize individual papers. They struggle with everything else. Single-prompt pipelines lose context across documents, overgeneralize from incomplete evidence, and produce outputs that are shallow and difficult to trust. At scale, they hallucinate patterns that do not exist.

This system was built to address that — through coordinated multi-agent reasoning, adaptive retrieval, and reliability-first engineering.

03 — Agents

What each agent does

01 — PLANNER
Planner Agent
Decomposes a vague topic into structured sub-questions and targeted search queries. Runs a self-evaluation inner loop — generated queries are assessed by a second LLM call and refined iteratively before retrieval begins.
query decompositionself-evaluationiterative refinement
02 — RESEARCH
Research Agent
Retrieves papers from arXiv and Semantic Scholar simultaneously. Evaluates retrieval quality against a minimum threshold and adaptively expands queries if insufficient papers are found. Applies deduplication and diversity control.
multi-sourceadaptive expansiondeduplication
03 — FILTER + RANK
Filtering & Ranking Layer
Scores papers using term-frequency relevance across title and abstract, combined with recency scoring via 1/(1+age). Ranked by a weighted formula: 0.6 × relevance + 0.4 × recency. Rejects papers with incomplete metadata.
relevance scoringrecency weightingtop-k selection
04 — SUMMARIZER
Summarizer Agent
Extracts structured knowledge from each paper: problem, methodology, key findings, limitations, core tradeoff, and confidence estimate. Validates output quality before accepting. Persists results to disk — re-runs skip already-computed summaries.
structured extractionconfidence calibrationfile caching
★ Core Intelligence Layer
05 — SYNTHESIS
Advanced Synthesis Agent
Where the system transitions from information extraction to genuine knowledge synthesis. Reasons across all papers simultaneously — not independently. Input is compressed to include only the fields needed for relational reasoning, reducing token overhead while preserving analytical depth.
Cross-paper reasoning Agreement detection Contradiction analysis Trend identification Research gap discovery Tradeoff analysis Future direction mapping Source-grounded insights
06 — CRITIC
Critic Agent
Evaluates synthesis on eight dimensions: reasoning depth, grounding quality, trend analysis, gap analysis, hallucination risk, specificity, diversity, and overall usefulness. Returns structured scores and flags weak or unsupported claims.
hallucination detectiongrounding evalself-critique
07 — FORMATTER
Formatter Agent
Converts structured synthesis into a professional literature review with sections for executive summary, comparative method analysis, research gaps, future directions, and conclusion. Academic tone, analytical depth, no repetition.
academic writingstructured reportmarkdown output
04 — Engineering

Reliability by design

In multi-stage agentic pipelines, infrastructure reliability becomes as important as model quality. Every agent is wrapped in the same set of guarantees.

⚡
File Caching
Summaries persist to disk. Re-runs skip computed stages entirely.
↩
Exponential Backoff
Randomized jitter on every retry. No API hammering under rate limits.
🔄
Adaptive Retrieval
Queries expand dynamically when paper quality falls below threshold.
🗂
State Persistence
Shared state accumulates across all agents. Any stage reruns independently.
✓
JSON Validation
Structured output validated at every stage before passing downstream.
⚙
DEV MODE
Lightweight execution path for fast, low-cost experimentation.
05 — Philosophy

Why framework-light

The architecture was built without heavy orchestration frameworks intentionally — to develop a deep understanding of state flow, retry logic, evaluation patterns, and agent interaction before abstracting them away. The goal is to understand the system, not just configure it.

Maps naturally to LangGraph-style orchestration — migration planned in a later phase.

07 — Limitations

Known constraints

Openly acknowledged — each maps to a specific next step.

Reasoning currently operates over abstracts rather than full papers — limiting deep experimental understanding and quantitative comparison.
Retrieval is lexical, not semantic. Conceptually similar papers with different terminology are under-recalled.
The Critic Agent evaluates synthesis quality but does not yet autonomously trigger refinement loops.
Evaluation metrics are heuristic rather than learned — synthesis quality scoring is still evolving.
No persistent cross-session memory yet — each run starts fresh without knowledge from previous queries.
08 — Vision

Where this is going

Semantic retrieval — embedding-based paper search replacing lexical overlap, dramatically improving recall for conceptually similar work with different terminology.
Critic-driven refinement loop — Critic scores below threshold trigger targeted re-runs of the Synthesis Agent with critique as context. True agentic self-improvement.
Persistent knowledge memory — reuse of previous research, summaries, and synthesis across sessions. JSON/SQLite first, then vector DB migration.
Full-paper reasoning — moving beyond abstract-only analysis toward deep experimental understanding and quantitative cross-paper comparison.
Citation-grounded synthesis — every insight traceable to a specific source, with citation-aware academic formatting in the final report.
LangGraph orchestration — migrating to a StateGraph with conditional routing, parallel execution, and retry nodes as first-class primitives.
09 — Motivation

Why I built this

I am interested in AI systems that do more than wrap a model — specifically in how multi-agent architectures handle tasks requiring planning, adaptive retrieval, cross-document reasoning, and self-evaluation.

One clear realization from building this: useful AI systems are not primarily about prompting. As systems scale, the hard problems become orchestration, reliability, state management, evaluation, and grounded reasoning. Those concerns become just as critical as model capability itself.

This project is an attempt to build that end-to-end — with reliability and grounded reasoning as first-class design goals, not afterthoughts.
